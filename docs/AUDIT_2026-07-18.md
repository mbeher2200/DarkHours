## 3. Confirmed discrepancies

Each entry: what the docs claimed → what the code actually does → where it was fixed.

### 3.1 Deployment model (major)

- **Claim (README):** the API runs as a Lambda **container image**; `Dockerfile`,
  `Dockerfile.lambda`, and `Dockerfile.worker` exist; CI "builds and pushes container
  images to ECR" before `cdk deploy`.
- **Reality (`cdk/lambda_api_stack.py`, `.github/workflows/deploy.yml`):** both the
  API and the worker are **zip Lambdas** (Python 3.13, arm64) whose dependencies are
  pip-installed by CDK asset bundling during deploy. There is no `Dockerfile` or
  `Dockerfile.lambda`. `Dockerfile.worker` exists only for the Trivy image scan in
  `security.yml` and for the throwaway in-region perf-test recipe in `CLAUDE.md`.
  `deploy.yml` is: pytest gate → GitHub OIDC → build SPA → `cdk deploy PyNightSkyLambda`.
  No Docker build, no ECR push.
- **Fixed in:** `README.md` (Cloud Deployment section), `CLAUDE.md` (clarifying note),
  `requirements-worker.txt` header comment.

### 3.2 Moonlight model (major)

- **Claim (README, PRODUCT.md):** moonlight interference uses the Krisciunas &
  Schaefer (1991) sky-brightness model.
- **Reality (`PyNightSkyPredictor/moonlight.py`):** since PR #111 the model is a
  **K&S 1991 × Winkler 2022 hybrid** — a single-scatter kernel with a two-component
  Rayleigh + Henyey-Greenstein (g = 0.8) phase function, driven by **live aerosol
  optical depth** (Open-Meteo CAMS) with a clear-sky fallback, plus an inverse-square
  Earth–Moon distance correction. `docs/PYNIGHTSKY.md` already documented this
  correctly; the top-level docs never caught up.
- **Fixed in:** `README.md`, `PRODUCT.md`.

### 3.3 Test suite (major)

- **Claims:** README: "423 tests". `CLAUDE.md`: "390 passed, ~7 skipped".
  `tests/README.md`: an 18-file inventory and a marker table without `live`.
- **Reality:** `pytest --collect-only -q` collects **803 tests across 36 files**;
  `pytest.ini` defines three markers: `eph`, `aws`, `live`.
- **Fixed in:** `README.md`, `CLAUDE.md`, `tests/README.md` (regenerated inventory).

### 3.4 Phantom `memory/` directory

- **Claim (`CLAUDE.md`, three references):** consult and update "the relevant
  `memory/` notes".
- **Reality:** no `memory/` directory exists in the repository.
- **Fixed in:** `CLAUDE.md` (references removed; `docs/` is the persistence layer).

### 3.5 Web app undocumented (major)

- **Claim (`apps/web/README.md`):** the stock "React + TypeScript + Vite" template
  text, with no mention of the actual application.
- **Reality:** `apps/web` is **DarkHours** (darkhours.app), a React 19 + TypeScript
  SPA and the product's primary surface — score card, night timeline with live "Now"
  row and WAQI haze cross-check, prime-target viability engine, 360° canvas sky dome
  with a 12k-star catalog and time scrubber, light-dome fisheye, Milky Way planner,
  aurora and meteor banners, satellite passes with Starlink trains, nearby dark-site
  finder with drive times, 30-day outlook heatmap, and a red night-vision mode.
- **Fixed in:** `apps/web/README.md` (full rewrite), `docs/FEATURES.md`, README's
  new web-app section.

### 3.6 Dependency attribution

- **Claim (`docs/ACKNOWLEDGMENTS.md`):** rasterio listed as a runtime Python
  dependency; fiona listed as a build-time dependency.
- **Reality:** rasterio appears only in `requirements-build.txt` (it reads the source
  GeoTIFFs when the raster grids are built; the runtime reads pure-numpy grid files —
  see 3.8). fiona was dropped entirely in PR #117.
- **Fixed in:** `docs/ACKNOWLEDGMENTS.md`, `requirements-worker.txt` header.

### 3.7 Trip no-weather weight redistribution

- **Claim (`docs/TRIPBUILDER.md`):** with `--no-weather`, "Lunar, Dark Hours, and
  Bortle are rescaled to 40%, 35%, 25%".
- **Reality (`PyNightSkyPredictor/scoring.py::rate_night`):** absent factors are
  renormalized **proportionally**: 25/25/10 → **≈41.7% / 41.7% / 16.7%**.
- **Fixed in:** `docs/TRIPBUILDER.md`.

### 3.8 Raster pipeline (initiative shipped, doc frozen at kickoff)

- **Claim (`docs/RASTERIO_REPLACEMENT.md`):** a plan to replace rasterio/GDAL
  (~335 MB image tax), status "starting context".
- **Reality:** shipped. The runtime path is `PyNightSkyPredictor/gridraster.py` —
  pure numpy tiled raw-binary grids read via local memmap or S3 byte-range requests.
  rasterio/GDAL survive only at build time (`gridbuild.py`, `requirements-build.txt`).
- **Fixed in:** `docs/RASTERIO_REPLACEMENT.md` (rewritten as the current pipeline
  doc), `README.md` (which claimed rasters are "Cloud-Optimized GeoTIFFs via GDAL"),
  `docs/ACKNOWLEDGMENTS.md`.

### 3.9 Nearby dark-sky search criteria

- **Claim (`docs/PYNIGHTSKY.md`):** candidates must be "at least 2 Bortle classes
  darker" than the origin and "no worse than Bortle 4"; the search "scans a grid of
  sample points (16 bearings × 11 rings)"; naming uses Overpass then Nominatim; up
  to 6 areas shown.
- **Reality (`darksky.py::_dark_threshold`, `find_nearby`):** the threshold is **one
  class darker, capped at Bortle 3** (a ≤ B2 origin requires B1) — the 2-class gap
  was deliberately relaxed because it starved already-dark origins. The search
  extracts dark pixels directly from the raster windows (land-masked, POI-first via
  the OSM index in the US); naming is a four-tier pipeline (POI index → PAD-US index
  → Overpass → reverse geocoder); up to 10 areas are shown.
- **Fixed in:** `docs/PYNIGHTSKY.md`.

## 4. Undocumented capabilities found in code

Features that existed in the codebase but appeared in no documentation (now covered
by `docs/FEATURES.md` and `apps/web/README.md`):

- **360° simulated sky dome** — canvas star field from a 12k-star HYG-derived
  catalog (`stars.v1.bin`), stereographic projection, drag-to-pan, sunset→sunrise
  time scrubber, real Milky Way texture, moon disc with phase, zodiacal light,
  limiting-magnitude/visible-star readout (`apps/web/src/report/skydome/`).
- **Prime-target viability engine** — per-target condition vectors (cloud,
  transparency, light dome, lunar proximity, meteor radiant altitude) rolled up to
  ok/degraded/blocked with effective imaging windows
  (`PyNightSkyPredictor/predictor.py::_apply_condition_vectors`).
- **Horizon light-dome analysis** — 8-direction Walker d^-2.5 kernel scores, dome
  distance and apparent height, all-sky fisheye panel, precomputed CONUS H3 index
  for instant page load (`light_dome.py`, `LightDomePanel.tsx`).
- **Aurora forecast** — NOAA SWPC Kp + centered-dipole geomagnetic-latitude viewline,
  three visibility tiers (overhead / naked-eye / photographic), look-direction
  bearing, moon/cloud/light-dome condition vector (`aurora.py`).
- **Meteor-shower activity model** — IMO log-linear ZHR decay with per-shower
  rise/decline constants, radiant-altitude local rate, limiting-magnitude
  degradation from the population index (`targets.py`).
- **Live WAQI haze cross-check** on the timeline "Now" row, with a 100 km station
  distance filter (`aqicn.py`).
- **Starlink train passes** (raising-phase filter) alongside ISS/Hubble/Tiangong
  (`satellites.py`, `tle_provider.py`).
- **Red night-vision mode** — full CSS-variable retheme to pure red channels plus an
  SVG `feColorMatrix` luminance→red filter for raster images (`02-red-mode.css`).
- **30-day outlook heatmap**, shareable permalinks, imperial/SI toggle, place
  autocomplete with recent-search history, CloudWatch RUM telemetry (`apps/web/src/`).
